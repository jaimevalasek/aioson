# Hardening Lane — From Idea to Executable Work

Hardening means reducing ambiguity, not multiplying documents.

## Canonical progression

1. Optional briefing frames an early idea.
2. Product creates one PRD.
3. Sheldon challenges and enriches that PRD in place.
4. Planner creates one vertical implementation plan.
5. Dev implements; QA proves the real production path.

Route back to Product/Sheldon when users, scope, observable behavior, failure states, or prototype intent are unclear. Route to one optional specialist only for a named decision that cannot responsibly be settled in the PRD or plan.

Hardening is complete when every required capability has a stable acceptance criterion and Planner can map it to exact paths and executable evidence. Requirements/spec/architecture/readiness/conformance/harness packs are not a measure of rigor.
