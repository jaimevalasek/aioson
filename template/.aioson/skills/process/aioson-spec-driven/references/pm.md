# Optional Specialist Reference — @pm

`@pm` is not in the canonical feature chain and does not own Gate C. `@planner` owns the implementation plan and Gate C.

Invoke PM only for a named backlog, prioritization, release, or stakeholder-coordination question that the reviewed PRD and implementation plan cannot express cleanly. Write the decision into the existing PRD or plan; do not create a parallel spec hierarchy or overwrite Planner's delivery phases.
