Padrão escalável para próximas releases

Cada nova versão segue o mesmo padrão: copiar a pasta tutorials/releases/1-9-0/ para tutorials/releases/<version>/ e adaptar o conteúdo. O template inclui hero com version-pill, métricas, timeline, evolução por tema (cards com snippets), what's next, CTAs e apêndice changelog colapsável.

Depois de criar a nova pasta, atualize este índice (releases/index.html) movendo a anterior do "latest" para o grid de histórico, e atualize o hub principal (tutorials/index.html) com a versão nova no featured-release.

3 lugares pra atualizar quando uma nova versão sair:

  1. tutorials/releases/<version>/index.html — copie a v1.9.0 e adapte
  2. tutorials/releases/index.html — promova ao latest, mova v1.9.0 pro grid
  3. tutorials/index.html — atualize featured-release no hub
