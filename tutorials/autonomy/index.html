3 tiers · permissões materializadas.

Contrato de autonomia em 3 níveis: tier1 silent (read), tier2 notified (memory writes), tier3 blocking (push/publish). permissions-generator materializa em allow-lists nativos de 4 harnesses no install. Tier3 é hard-rejected mesmo se um tool listar — ações irreversíveis sempre exigem mão humana.

3 tiers · escalating risk

Cada tier tem uma resposta diferente.

Decisão de UX: leituras passam silenciosamente, escritas em memória notificam, escritas irreversíveis bloqueiam. Cada agente sabe em qual tier opera; harness enforce.

1
silent

Tier 1 · read

Leituras + telemetria. Não requer notificação. Agentes consultam livremente — não há impacto observável pelo usuário. Sempre permitido.

Exemplos preflight · memory:status brain:query · context:load memory:search · gate:check
2
notified

Tier 2 · memory writes

Mutações em memória (rules, brains, learnings). Notifica via aioson notify --level=warn antes da mutação. Usuário vê a ação acontecer — preserva accountability sem bloquear automation.

Exemplos memory:archive · memory:restore memory:reflect-commit learning:auto-promote
3
blocking

Tier 3 · irreversible

Operações irreversíveis ou com impacto público. Hard-rejected em qualquer harness, regardless of allow-list config. Sempre exigem ação humana explícita fora do agent loop.

Exemplos git push · npm publish cloud:publish:squad genome:publish · skill:publish

decision-presentation skill · 3 profiles

Adapta a interação ao usuário.

Campo profile em project.context.md define como agentes user-facing (@neo, @setup, @product, @dev, @deyvin) interagem. Default seguro: creator. Migração automática beginner → creator em aioson update.

creator

Usuário leigo · default

Para quem está construindo sem ser dev. Tudo guiado, sem jargão, com tempo pra pensar.

  • 1 pergunta por turn via AskUserQuestion
  • (Recomendado) marker obrigatório
  • Jargão traduzido por dicionário
  • "Pausar / quero pensar" sempre presente
developer

Dev solo

Para devs que sabem o que querem. Cadência rápida, jargão permitido, batches de perguntas.

  • Até 5 perguntas numbered por batch
  • Jargão permitido sem tradução
  • (Recomendado) marker opcional
  • Free-form questions permitidas
team

Time com stakeholders

Operadores leem jargão; stakeholders leem tradução. Executive summary auto-gerado.

  • Cadência igual a developer
  • Jargão no flow operacional
  • Executive summary em agent:done
  • Dicionário aplicado no summary
