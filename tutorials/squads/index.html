O que é um Squad: diferente de invocar agentes oficiais um a um, um squad é um time persistente e configurável com manifesto próprio, executores customizados, memória/aprendizado compartilhados, comunicação interna (bus/webhook), telemetria de ROI e ciclo de vida (publish/install/deploy). Você cria squads para domínios específicos: software, marketing, jurídico, suporte, comercial, criação de conteúdo.

Anatomia de um squad

Os 3 elementos básicos.

Todo squad tem agentes (quem decide), executores (quem faz) e infraestrutura (como se comunicam e aprendem). O resto é configuração.

Agentes (decisores)

Personalidades com tier 0-3, mission, DISC profile e domínio. Definem o quê fazer. Podem ser oficiais (@product, @dev) ou customizados por squad:agent-create.

Executores (workers)

Workers determinísticos que recebem tasks do agente. Tipos: shell, http, llm, mcp, custom. Rodam via squad:daemon ou squad:worker individual.

Infraestrutura

Bus de mensagens intra-squad, webhook externo, MCP servers, dashboard de ROI, memória + learning loop. Tudo configurável no manifesto + comandos dedicados.

Tutorial · primeiro squad

Crie e rode um squad em 5 passos.

Do scaffold ao dashboard. Cada passo é um comando — squads são primeira-classe no CLI, não exigem GUI.

1

Scaffold da estrutura base

Cria .aioson/squads/{slug}/ com manifesto, executors vazios, contracts iniciais.

scaffold
$ aioson squad:scaffold . --name=marketing-team --domain='content' ✓ created .aioson/squads/marketing-team/ · manifest · 0 agents · 0 workers
2

Adicionar agentes ao squad

Use agentes oficiais ou crie customizados. Tier 0-3 controla autonomia, DISC profile define personalidade.

agent-create
$ aioson squad:agent-create . --squad=marketing-team \\ --name=copywriter --type=agent --tier=2 --disc=I \\ --mission='Write conversion-focused copy' --domain='marketing' ✓ agent copywriter created · tier 2 · DISC=I
3

Validar manifesto + dependências

Verifica schema, dependency graph e contratos antes de rodar. Quebra cedo, não no daemon.

validate + dependency-graph
$ aioson squad:validate . --squad=marketing-team ✓ manifest valid · 1 agent · 0 unresolved deps $ aioson squad:dependency-graph . --squad=marketing-team → copywriter (entry) · no downstream
4

Subir o daemon e dashboard

Daemon escuta eventos, dashboard mostra telemetria em real-time (porta 4180 por padrão).

daemon + dashboard
$ aioson squad:daemon . --sub=start --squad=marketing-team ✓ daemon started · pid 12847 · port 4180 $ aioson squad:dashboard . --squad=marketing-team --port=4180 → http://localhost:4180/squad/marketing-team
5

Disparar uma task + monitorar

Send a task via bus, monitor execution, track ROI metrics.

bus + status + roi
$ aioson squad:bus . --squad=marketing-team --to=copywriter \\ --message='Write landing hero for new SaaS launch' ✓ message dispatched · waiting... $ aioson squad:roi . --sub=report --squad=marketing-team tasks: 1 · avg_time: 4.2s · tokens: 1.8k · cost: $0.04
Casos de uso

Squads por domínio.

Cada squad é um time inteiro. Reusa agentes oficiais quando fazem sentido, cria customizados pro que é específico do domínio.

software

Dev squad full-stack

Time completo de software com SDD: PRD → arquitetura → implementação → QA → security audit.

  • @product · @analyst · @architect
  • @dev · @qa · @pentester
  • workers: shell (npm), http (CI)
  • bus: webhook GitHub
marketing

Content marketing squad

Pesquisa, copy, edição, distribuição. Genomes de copywriter (Brunson, etc) injetados por padrão.

  • @orache (research) · @copywriter
  • custom: editor · custom: scheduler
  • workers: http (CMS), mcp (analytics)
  • roi: pageviews / conversões
jurídico

Legal review squad

Triagem de contratos, redação de cláusulas, compliance check, sumário executivo.

  • custom: triador · custom: redator
  • custom: compliance-officer
  • workers: shell (LLM local), mcp (notion)
  • memória: precedentes do escritório

Categoria 1 · 6 comandos
Criar (compose)

Composição do squad: scaffolding inicial, criação de agentes customizados, planejamento de pipeline, mapeamento de dependências, card de apresentação e validação do manifesto.

aioson squad:scaffold
Estrutura inicial
Cria .aioson/squads/{slug}/ com manifest, contracts, executors vazios, README.
aioson squad:agent-create
Agente customizado
--type=agent|assistant|clone|worker --tier=0|1|2|3 --disc=<profile>. Opcional --with-infra cria scaffold de workers.
aioson squad:plan
Plano de execução
Gera plan.json do squad: ordem dos agentes, gates internos, dependências, tier mapping.
aioson squad:dependency-graph
Grafo de dependências
Visualiza quem chama quem dentro do squad. Detecta ciclos. Usa antes de daemon.
aioson squad:card
Card público
Gera card de apresentação do squad pra marketplace/registry. Title, summary, capabilities.
aioson squad:validate
Schema check
Valida manifest, schemas, dependency graph e contratos. Quebra cedo — antes do daemon.
criar
$ aioson squad:scaffold . --name=dev-team --domain=software ✓ .aioson/squads/dev-team/ created · 0 agents · manifest seeded $ aioson squad:agent-create . --squad=dev-team --name=tech-lead --type=agent --tier=2 ✓ agent tech-lead added · DISC=D · tier 2 $ aioson squad:validate . --squad=dev-team ✓ manifest valid · 1 agent · 0 unresolved deps
Categoria 2 · 6 comandos
Executar (run)

Tudo de execução: daemon (long-running), autorun (one-shot), workers individuais, processos rodando, worktrees git paralelas, e recovery de crashes.

aioson squad:daemon
Long-running daemon
--sub=start|status|stop|logs. Escuta bus + webhook, distribui tasks pros workers. Porta padrão 4180.
aioson squad:autorun
One-shot execution
Roda o squad inteiro até completar a task atual, sem ficar de daemon. Bom pra CI.
aioson squad:worker
Worker individual
--sub=list|run|test|logs|scaffold. Roda 1 worker isolado pra debug ou tarefa pontual.
aioson squad:processes
Inspect processos
Lista todos os processos do squad ativo (daemon, workers, MCP). PID, memória, uptime.
aioson squad:worktrees
Git worktrees paralelas
Cria worktrees git pra cada lane do squad rodar isolada. Evita conflito de branch.
aioson squad:recovery
Recovery após crash
Restaura estado do squad a partir do último checkpoint válido. Limpa locks órfãos.
executar
$ aioson squad:daemon . --sub=start --squad=dev-team ✓ daemon · pid 12847 · listening on :4180 $ aioson squad:processes . --squad=dev-team daemon: 12847 · workers: 12849 (ci-runner), 12851 (linter) · mcp: 12853
Categoria 3 · 7 comandos
Coordenar (orchestrate)

Comunicação interna (bus) e externa (webhook), merging de output de várias lanes, deploy do squad inteiro, pipelines pré-definidos, ciclo de aprendizado e review entre agentes.

aioson squad:bus
Mensagens intra-squad
Dispatch de mensagens entre agentes do squad. --to=<agent> --message=<text>. Substituto moderno do mailbox.
aioson squad:webhook
Webhooks externos
Recebe POSTs externos (GitHub, Slack, etc) e dispara tasks no squad. URL emitida pelo daemon.
aioson squad:merge
Merge de output
Combina output de várias lanes paralelas em um único resultado. Strategies: concat, dedup, vote.
aioson squad:deploy
Deploy do squad
Empacota o squad inteiro e deploya em um ambiente alvo (cloud, server, etc). Versioned.
aioson squad:pipeline
Pipelines nomeadas
--sub=list|show|status --pipeline=<slug>. Pipelines pré-configuradas dentro do squad.
aioson squad:learning
Ciclo de aprendizado
Roda o learning loop interno do squad: distillation de tasks recentes em rules/brains próprias.
aioson squad:review
Review entre agentes
Faz um agente revisar output de outro antes de aprovar. Útil pra QA automation interno.
coordenar
$ aioson squad:bus . --squad=dev-team --to=tech-lead --message='New ticket: optimize query' ✓ message id=msg-abc · dispatched · awaiting ack $ aioson squad:pipeline . --sub=show --squad=dev-team --pipeline=ci-on-pr stages: lint → tests → security-audit → deploy-preview
Categoria 4 · 6 comandos
Distribuir (share)

Publicar squads no registry da nuvem, instalar squads de outros, dar acesso via grant, exportar snapshot local, reparar genomes corrompidos, listar squads disponíveis.

aioson squad:publish
Publicar no registry
--bundle-skills=a,b --bundle-genomes=a,b --version=1.0.0 [--private] [--paid]. Versionado.
aioson squad:install
Instalar squad
<code-or-slug> [--email]. Baixa do registry e configura local. Para squads pagos: email + grant.
aioson squad:grant
Conceder acesso
<code> <email>. Owner concede acesso a um squad pago para um usuário específico.
aioson squad:export
Snapshot local
Exporta squad em tar.gz para distribuição manual ou backup. Inclui manifests, brains, contracts.
aioson squad:repair-genomes
Reparar genomes
<manifest.json>. Repara genomes do squad com schema antigo ou paths quebrados. Backup automático.
aioson squad:list
Listar squads
[--remote]. Lista squads locais ou navega no registry remoto.
distribuir
$ aioson squad:list . --remote found: 47 squads · 12 free · 35 paid $ aioson squad:install . marketing-pro --email=user@example.com ✓ installed marketing-pro@2.1.0 · 4 agents · 3 genomes bundled
Categoria 5 · 6 comandos
Observar (monitor)

Dashboard web em real-time, status snapshot, doctor com auto-fix, scoring de tasks, ROI tracking com métricas customizadas, e investigações estruturadas.

aioson squad:dashboard
Dashboard web
--port=4180 --squad=<slug>. Telemetria em real-time: tasks, mensagens, ROI, processos.
aioson squad:status
Status snapshot
CLI snapshot: daemon up, agents idle/busy, queue depth, last task. Ideal para preflight.
aioson squad:doctor
Health check
--stale-minutes=5. Detecta workers stale, locks órfãos, dependências quebradas. Sugere fixes.
aioson squad:score
Score de tasks
Pontua tasks executadas por métricas configuráveis. Persistido em runtime SQLite para histórico.
aioson squad:roi
ROI tracking
--sub=config|metric|report|export --key=<m> --value=<n>. Métricas próprias: tokens, $, tempo, leads.
aioson squad:investigate
Investigações
--sub=list|show|score|link|register --investigation=<slug>. Issues, RCAs, post-mortems do squad.
observar
$ aioson squad:status . --squad=dev-team daemon: up (2h 14m) · workers: 3 idle, 1 busy · queue: 0 $ aioson squad:roi . --sub=report --squad=dev-team last 7d · tasks: 142 · tokens: 1.8M · cost: $42.30 · avg_time: 2.4min
Categoria 6 · 3 comandos
Estender (extend)

MCP (Model Context Protocol) servers integrados ao squad pra acessar ferramentas externas (filesystem, GitHub, Notion, etc) e tools customizadas registráveis. Esta é a camada de extensibilidade.

aioson squad:mcp
MCP server manager
--sub=status|connectors|configure|test --mcp=<slug>. Gerencia servers MCP do squad.
aioson squad:mcp:call
Chamar tool MCP
Invoca uma tool específica de um connector MCP do squad. Testa integração antes de wire no agente.
aioson squad:tool:register
Registrar tool custom
Registra função customizada (JS/Python/shell) como tool acessível pelos agentes do squad.
estender
$ aioson squad:mcp . --sub=status --squad=dev-team connectors: github (✓), notion (✓), slack (offline) $ aioson squad:tool:register . --squad=dev-team --tool=deploy-preview --executor=shell ✓ tool deploy-preview registered · available to all agents
