O problema antes do v1.9.0: toda sessão começava com agente lendo 4-6 arquivos de contexto pra entender onde parou — projeto context, spec ativa, requirements, plano. Reflect engine · 5 passos

Como a memória se auto-atualiza.

Após cada agent:done ou workflow:next --complete, o engine analisa o git diff e decide se justifica atualizar os bootstrap files. Tudo deterministicamente.

01
agent:done

Trigger

Agente fecha sessão. Hook em runAgentDone chama memory:reflect-prepare.

02
classify diff

Classify

Engine puro (src/memory-reflect-engine.js) analisa diff vs sinais: routes, models, contratos, volume.

03
manifest

Write manifest

Se relevante: escreve .aioson/runtime/reflect-prompt.json listando targets, snapshot hash, allowed paths.

04
agent reflects

Agent edits

Próxima sessão recebe ⚠ no prompt. Agente edita os targets respeitando frontmatter intact.

05
commit

Persist

memory:reflect-commit valida (snapshot hash, allowed paths, frontmatter) e persiste em disco.

reflect commit

Doctor · 3 checks de curadoria

A memória se auto-fiscaliza.

Esses checks rodam dentro de aioson doctor com severity warning. Não bloqueiam o build, mas surfaceiam quando algo está envelhecendo.

rule_staleness

Rules em .aioson/rules/ que não foram carregadas (rule_loaded event) nas últimas N features. Threshold dinâmico: max(5, ceil(avg_days_between_last_5_features / 7)).

living-memory:rule_staleness

learning_orphans

Learnings status='promoted' cuja rule não tem rule_loaded event posterior ao updated_at da promoção. Sinal de que o pattern foi promovido mas ninguém usa.

living-memory:learning_orphans

distillation_lag

Features status=done sem entry auto_distillation em evolution_log. Fires apenas quando ≥ 5 features fecharam — evita falso positivo em projetos novos.

living-memory:distillation_lag
