Categoria 1 · 12 scripts
Spec-Driven core

A fundação do AIOSON. Cobrem pré-voo, classificação, gates A·B·C·D, artefatos, telemetria e fechamento de feature.

aioson preflight
Pré-voo completo
Modo, classificação, framework, test runner, artefatos, gates e prontidão em uma chamada.
aioson classify
MICRO / SMALL / MEDIUM
Scoring automático via PRD + requirements.
aioson sizing
Modelo de sizing
Inplace, phased_inplace, phased_external contando entidades e integrações.
aioson detect:test-runner
Detecção do runner
PHPUnit, Jest, Vitest, Pytest, RSpec, Forge, node:test por arquivos de config.
aioson gate:check
Phase gate
Valida pré-requisitos e artefatos dos gates A/B/C/D. PASS ou BLOCKED.
aioson artifact:validate
Cadeia de artefatos
PRD → spec → plano → conformance e indica o próximo item faltante.
aioson pulse:update
Pulse update
Cria ou atualiza project-pulse.md com agente, feature, gate e próximo passo.
aioson state:save
Ponto de continuação
Alias dev:state:write. Fase, status e spec-version em dev-state.md com histórico.
aioson feature:close
Fechamento
Atualiza spec (QA sign-off), features.md e pulse. Dispara distillation hook.
aioson workflow:execute
Orquestração
Monta e executa o plano de agentes por tier. --dry-run e --start-from.
aioson workflow:next
Próximo estágio
Avança o workflow. --complete, --status, --suggest, auto-routing.
aioson learning:auto-promote
Promoção de aprendizados
Promove patterns process/quality (threshold ≥ 3) para .aioson/rules/.

aioson memory:archive
Tier-2 archive

aioson memory:restore
Restaurar archived

Reflect manifest
Classifica diff pós-agent e escreve .aioson/runtime/reflect-prompt.json.
aioson memory:reflect-commit

Categoria 5 · 8 scripts · feature-dossier
Dossier

Cada feature SMALL/MEDIUM ganha um .aioson/context/features/{slug}/dossier.md consolidando Why/What + code map + agent trail. Auto-init silencioso após PRD. Consultado por @dev/@deyvin pra detectar drift. É a "memória da feature" — independente da memória global do projeto.

aioson dossier:init
Criar dossier
Init silencioso por @product. Cria features/{slug}/dossier.md com Why/What.
aioson dossier:show
Ler dossier
Exibe conteúdo do dossier de uma feature. Usado por @dev em cold start.
aioson dossier:add-finding
Adicionar finding
--section="Agent Trail" --content="Slice: X". Cada agente registra seu pass.
aioson dossier:add-codemap
Mapear arquivos
Registra paths que cada slice de implementação tocou. Usado pra drift detection.
aioson dossier:link-rule
Linkar rule
Associa .aioson/rules/X.md consultadas durante a feature ao dossier.
aioson dossier:add-research
Adicionar research
Vincula items de researchs/ consumidos durante discovery/architect.
aioson dossier:audit
Audit do dossier
Valida completude (Why/What presentes, Agent Trail tem entradas, code_map populado).
aioson dossier:compact
Compactar dossier
Resume entries antigos do Agent Trail mantendo decisões importantes. Reduz tamanho.

Categoria 6 · 3 scripts · @deyvin
Scout (sub-task)

Surveys context-isolated. Quando @deyvin precisa investigar > 5 arquivos ou trace runtime, dispatch um scout em vez de carregar tudo no parent. Resultado volta sumarizado — pai não queima contexto.

aioson scout:prep
Preparar scout
Monta o prompt para sub-agente read-only com parent_session_excerpt + scope_paths. Caps por sessão/scope.
aioson scout:validate
Validar output
Verifica JSON schema do output do scout, aplica caps, retorna PASS ou retry.
aioson scout:commit
Commit do scout
Persiste em .aioson/runtime/scouts/, emite telemetry sub_task. Auto-archive on feature:close.
scout · @deyvin dispatch
$ aioson scout:prep . --topic='auth flow trace' --scope-paths=src/auth,src/middleware scout_id: scout-abc123 · caps: 50 files max · 3 retries $ aioson scout:validate . --scout-id=scout-abc123 --input=output.json ✓ schema valid · 18 files visited · within caps $ aioson scout:commit . --scout-id=scout-abc123 --feature=checkout ✓ committed · will archive to features/checkout/scouts/ on close
Categoria 7 · 3 scripts · MEDIUM features
Harness validator

Gate D enforcement via JSON output. @sheldon gera harness-contract.json com binary criteria, @validator avalia headless, circuit breaker bloqueia closure se ainda houver erros. Só MEDIUM — opt-in.

aioson harness:init
Inicializar contrato
Cria .aioson/plans/{slug}/harness-contract.json com schema vazio pra @sheldon popular.
aioson harness:validate
Router de validação
Se last-validator-output.json existe, aplica via apply-validation; senão gera prompt headless.
aioson harness:apply-validation
Consumir output
Traduz results[].reason do primeiro fail em progress.json.last_error. Circuit breaker.
harness · ciclo
