# Requirements — Operator Memory

> **Reading discipline (BR-AN-01):** This document does NOT duplicate the 58 per-phase ACs already defined in `.aioson/plans/operator-memory/plan-*.md`. It consolidates cross-cutting requirements, NFRs, entity field-level schemas, edge cases, migration order, and the 2 analyst-deferred decisions (DD-08, DD-09). Per-phase ACs are referenced by ID, not copied.

## Feature summary

Per-operator decision-memory layer multi-dev safe via sha256 git-email hash, harness-agnóstica. 5 phases progressive release v1.12.0 → v1.16.0 (DD-05 from sheldon). Storage: hybrid SQLite (FTS5 index) + markdown (body). LLM-driven capture acknowledged divergence from AIOSON's deterministic principle (PMD-02).

## Entities (Phase 3 field-level)

### `operator_identity` (resolved-only — no on-disk schema)

| Field | Type | Nullable | Constraints |
|-------|------|----------|-------------|
| identity | string | no | sha256(email)[0..16] OR `AIOSON_OPERATOR_ID` value matching `^[a-z0-9][a-z0-9-]{2,31}$` |
| source | enum | no | `email-hash` \| `override` \| `anonymous-fallback` |
| storage_root | string | no | absolute path to `~/.aioson/operators/{identity}/` |

### `_index.sqlite` schema (PMD-01 hybrid storage)

**Table `operators`** — registry of seen identities (one row per `~/.aioson/operators/{identity}/`):

| Field | Type | Nullable | Constraints |
|-------|------|----------|-------------|
| identity | TEXT | no | PRIMARY KEY |
| created_at | TEXT (ISO) | no | inserted on first activity |
| source | TEXT | no | `email-hash` \| `override` \| `anonymous-fallback` |
| last_active_at | TEXT (ISO) | no | updated on each op:* operation |

**Virtual table `decisions_fts`** (FTS5):

| Field | Type | Notes |
|-------|------|-------|
| identity | UNINDEXED | partition key — queries filter by identity first |
| slug | UNINDEXED | join key back to markdown file |
| signal_type | INDEXED | for filter `WHERE signal_type='authorization'` |
| category | INDEXED | for filter by category (identity/autonomy/tooling/default) |
| body | INDEXED | porter tokenizer — full-text search on decision body |
| last_reinforced | UNINDEXED | for decay queries (loaded into memory; FTS5 lacks range scan) |

**Recommended indexes:**
- `decisions_fts` natural indexes on tokenized columns (FTS5 built-in)
- Cover index for decay sweep: in-memory scan of `last_reinforced` by category (small N per identity — 10k cap)

### `proposal` (markdown at `~/.aioson/operators/{identity}/proposals/{slug}.md`)

**Frontmatter schema:**

| Field | Type | Nullable | Constraints |
|-------|------|----------|-------------|
| slug | string | no | derived from `proposal` via deterministic kebab-case + truncate 40 + collision suffix |
| signal_type | enum | no | `authorization` \| `exclusion` \| `correction` \| `confirmation` |
| detected_count | int | no | starts at 1; promote at ≥ 2 (PMD-07) |
| first_detected | timestamp (ISO) | no | UTC |
| last_detected | timestamp (ISO) | no | UTC, updated each re-detection |
| quotes | string[] | no | verbatim user messages, capped at 5 most recent (AC-AN-07) |
| proposal | string | no | paraphrase generated by LLM at capture time |
| source_agent | string | no | which agent observed the signal (`dev`, `deyvin`, etc.) |

**Body:** none (frontmatter-only file).

### `decision` (markdown at `~/.aioson/operators/{identity}/decisions/{slug}.md`)

**Frontmatter schema (resolves DD-08):**

| Field | Type | Nullable | Constraints |
|-------|------|----------|-------------|
| slug | string | no | same as proposal slug |
| signal_type | enum | no | same enum as proposal |
| promoted_at | timestamp (ISO) | no | UTC, moment of promotion |
| last_reinforced | timestamp (ISO) | no | updated on each re-detection post-promotion (PMD-11) |
| reinforcement_count | int | no | starts at 0 at promote (0 reinforcements yet); ++ on each post-promote re-detection |
| superseded_by | string | yes | slug of replacement decision (nullable; populated by V2 when one decision invalidates another) |
| category | enum | no | `identity` \| `autonomy` \| `tooling` \| `default` (PMD-03) |
| source_agent | string | no | agent that triggered promotion |
| quotes | string[] | no | first 5 trigger quotes preserved from proposal (audit trail) |
| version_schema | string | no | "1.0" — for V1→V2 migration; bumped on breaking schema changes |
| deprecated_by | string | yes | reverse pointer when this decision was the `superseded_by` target (V2 marker, nullable) |

**Body:** human-readable summary, ≤ 500 chars (AC-AN-08), markdown allowed. Format:
```markdown
# {Title — derived from proposal}

{One paragraph in user-facing language describing the decision.}

## Trigger quotes
- "{quote 1}"
- "{quote 2}"
- ...
```

### `conflict_warning` (transient, not persisted — emitted at preflight)

| Field | Type | Nullable | Constraints |
|-------|------|----------|-------------|
| decision_slug | string | no | references decision that conflicts |
| rule_path | string | no | absolute path to project rule |
| severity | enum | no | `warning` V1 (no `error` mode — V2 escalation) |
| reason | string | no | one-line summary of the keyword overlap that triggered detection |

Persistence: not on disk. Emitted to stderr per AC-P4-02; debounced via `~/.aioson/operators/{identity}/_conflict_state.json` (last_warned_at per slug).

### Auxiliary state files

- `~/.aioson/operators/{identity}/_decay_state.json` (Phase 5) — `{slug: last_decay_prompt_at_ISO}` per slug, debounce 30d window
- `~/.aioson/operators/{identity}/_conflict_state.json` (Phase 4) — `{slug: last_conflict_warning_at_ISO}` per slug-rule pair, debounce 60s window
- `~/.aioson/operators/{identity}/history/{ISO}-{slug}.md` (Phase 2+) — soft-deleted decisions/proposals, hard-deleted by Phase 5 cleanup after 365d

## MEMORY.md format spec (resolves DD-09)

**`~/.aioson/operators/{identity}/MEMORY.md`:**

```markdown
---
identity_prefix: {first 8 chars of identity}
decisions_count: N           # decisions in this file (active tier)
archived_count: M            # decisions in MEMORY-archive.md (lazy-load)
last_promoted: {ISO}
schema_version: "1.0"
---

# Operator Memory — Index

## Active decisions

- [Commit autonomy after slice approval](decisions/commit-autonomy-after-slice.md) — authorization, reinforced 2026-05-21
- [npm publish stays manual](decisions/npm-publish-manual.md) — exclusion, reinforced 2026-05-20
- ...

## See also

- `MEMORY-archive.md` — {M} archived decisions ({last_archival_at})
```

**Tier-based scaling rule (DD-09 resolution):**

1. **Active tier (in MEMORY.md):** decisions where `(now - last_reinforced) < category_half_life`. Auto-loaded by preflight directive.
2. **Archive tier (in MEMORY-archive.md):** decisions where `(now - last_reinforced) ≥ category_half_life` AND not yet pruned. Lazy-load only on explicit `op:show <slug>` OR `op:list --include-archived`.
3. **Pruned tier (in history/):** decisions removed by Phase 5 hard cap or by `op:forget`. Not searchable from MEMORY.md.

**Migration trigger:** decay sweep (Phase 5) moves decisions from active → archive on category-half-life crossing. `op:reinforce` moves back from archive → active. Format identical between MEMORY.md and MEMORY-archive.md (just different files).

**Why tier-based vs pagination/truncation:**
- Pagination across multiple files breaks lazy-load locality (preflight would need to discover N files).
- Truncation loses information silently — anti-pattern.
- Tier-based preserves audit trail + keeps active set bounded to current-relevance decisions. Aligns with Engram's "recent + reinforced first" heuristic.

**Active-tier size bound:** with per-category half-life worst case (identity=365d), active tier per identity ≤ ~365 days × ~1 promote/week ≈ 50 decisions typical. Hard cap 10k applies across active+archive combined; pruning at hard cap targets non-identity oldest from archive first.

## Cross-cutting non-functional requirements (NFRs)

### NFR-01 — Storage scaling

| Sub-AC | Requirement |
|--------|-------------|
| AC-NFR-01-a | Preflight load (read MEMORY.md only) ≤ 50ms cold on machine with 10k decisions across identities, on local SSD. |
| AC-NFR-01-b | FTS5 query `SELECT slug FROM decisions_fts WHERE identity=? AND decisions_fts MATCH ?` ≤ 20ms for 1k decisions in active identity. |
| AC-NFR-01-c | `_index.sqlite` size ≤ 50 MB at hard cap (10k decisions × ~5KB body each). |
| AC-NFR-01-d | WAL mode enabled; `PRAGMA integrity_check` runs clean after 1000 op:capture/op:promote operations in fixture stress test. |

### NFR-02 — Prompt budget

| Sub-AC | Requirement |
|--------|-------------|
| AC-NFR-02-a | Universal loading directive (Phase 3) ≤ 300 tokens per agent file (AC-P3-06). |
| AC-NFR-02-b | Cross-cutting overhead across all template agent files (~30) ≤ 6k tokens added by directive injection. Measured via `scripts/memory-budget-audit.js`. |
| AC-NFR-02-c | Lazy-loaded decision (single `decisions/*.md`) ≤ 1k tokens per file (body ≤ 500 chars + frontmatter ≤ 500 chars). |
| AC-NFR-02-d | Active-tier MEMORY.md ≤ 5KB per identity (50 decisions × ~100 chars/line). |

### NFR-03 — Multi-dev isolation

| Sub-AC | Requirement |
|--------|-------------|
| AC-NFR-03-a | Two operators on shared `~` (rare but possible — shared host): zero cross-read between identities. Fixture proves Alice's `op:list` shows only Alice's decisions; Bob's only Bob's. |
| AC-NFR-03-b | Same operator on different hosts but same repo (git share): zero git-side leak. `~/.aioson/` is never committed; PRD out-of-scope confirms machine-local V1. |
| AC-NFR-03-c | Identity-hash collision probability ≤ 10^-9 for teams ≤ 1000 emails (sha256[0..16] gives 2^64 space — meets bound). |
| AC-NFR-03-d | `AIOSON_OPERATOR_ID` override is per-process, per-session — never persisted (V1; `op:identity set` in Phase 5 is session-scoped). |

### NFR-04 — Decay correctness

| Sub-AC | Requirement |
|--------|-------------|
| AC-NFR-04-a | Per-category half-life thresholds applied exactly: identity=365d, autonomy=180d, tooling=90d, default=90d (PMD-03). |
| AC-NFR-04-b | Decay prompt debounced 30d per slug — same slug not surfaced twice within window. |
| AC-NFR-04-c | False positive rate in fixture corpus (15 stable + 10 should-decay decisions): ≤ 1 FP per category per 90-day window. |
| AC-NFR-04-d | `op:reinforce` resets `last_reinforced` to now without modifying any other field. |

## Migration order (entity creation across phases)

1. **Phase 1 (v1.12.0):** create `~/.aioson/operators/` root + `_index.sqlite` (schema: `operators` table only; FTS5 virtual table created but unused). Identity resolution operational.
2. **Phase 2 (v1.13.0):** activate FTS5 virtual table (mirror writes from `op:capture` / `op:promote`). Create `proposals/` + `decisions/` directories on first use.
3. **Phase 3 (v1.14.0):** create `MEMORY.md` index on first decision write. No schema migration (additive only).
4. **Phase 4 (v1.15.0):** create `_conflict_state.json` on first conflict warning. Project rule schema additive: `conflicts_with_signal_types` optional field.
5. **Phase 5 (v1.16.0):** create `_decay_state.json` on first decay sweep. `MEMORY-archive.md` created on first archive event. `user-profile.md` migration adds `deprecated_by` + `deprecated_at` frontmatter fields.

**No destructive migrations across phases.** All schema evolution is additive (new fields, new files) — backward-compat with prior phase outputs. V1 → V2 schema migrations (per `version_schema` field) are out of scope for this MEDIUM feature.

## Critical business rules (BRs)

- **BR-AN-01** Phase-plan ACs (58 total) are normative; this document does NOT duplicate them. Per-phase implementation, testing, and Gate D verification reference plan files directly.
- **BR-AN-02** Identity resolution is non-blocking. If git config absent OR email empty: fall back to `_anonymous` reserved identity, log telemetry `op_identity_unresolved`, continue (do NOT hard-error).
- **BR-AN-03** Capture is best-effort. If `op:capture` invocation fails (CLI exits non-zero), the agent session must NOT crash — capture is informational, not critical-path.
- **BR-AN-04** Promotion is atomic (AC-P2-03). No partial states observable — crash mid-promote leaves either both old (proposal + no decision) or both new (decision + no proposal) on next CLI invocation.
- **BR-AN-05** Project rules ALWAYS win on conflict (PMD-09). Operator decision is never modified by conflict detection — only stderr warning emitted + telemetry. User remains free to `op:forget` the conflicting decision if they want to silence it.
- **BR-AN-06** Hard cap 10k per identity is enforced at promote-time. Pruning preserves identity-category decisions (never auto-pruned). Pruned items move to `history/` (soft-delete) before hard-delete at 365d (Phase 5 cleanup).
- **BR-AN-07** `last_reinforced` updates on capture-re-detection AND `op:reinforce` (PMD-11). Updates from conflict warnings do NOT count (warning is not reinforcement signal).
- **BR-AN-08** Tier transitions (active ↔ archive) are reversible by `op:reinforce`. Tier transition does NOT modify decision body — only `last_reinforced` timestamp.
- **BR-AN-09** Telemetry events emit BEFORE file writes (consistent with F2 pattern from workflow-handoff-integrity). Crash post-write leaves auditable trail; crash post-telemetry-pre-write is observable as orphan telemetry.
- **BR-AN-10** Feature flag `AIOSON_OPERATOR_MEMORY` (Phase 3) default OFF until Phase 4 ships green; default ON post-Phase-4 (per inception risk mitigation in sheldon manifest).

## Edge cases

- **EC-01 — Empty git email:** dev runs without `git config user.email` set. Identity falls to `_anonymous` (reserved exception to PMD-05 reserved-prefix rule). Telemetry warns. Decisions accumulate under `_anonymous`. Multi-dev scenarios: ALL anonymous users share a single bucket — this is a documented limitation (V1 acceptable since git-email-less devs are rare).
- **EC-02 — Email change (rebase identity):** Alice changes `git config user.email` after accumulating decisions. New hash → new identity. Old decisions remain under old hash, unreachable. **Mitigation:** `aioson op:identity set <old-hash>` in Phase 5 allows pointing at old identity manually. Document in user guide.
- **EC-03 — Concurrent `op:capture` from parallel agents:** two agents in same session emit `op:capture` for similar signals quasi-simultaneously. SQLite WAL handles concurrency; deterministic slug derivation ensures the second arrival sees `detected_count: 1` and increments to 2 — promoting earlier than would otherwise (this is desired: rapid same-session reinforcement is signal).
- **EC-04 — `_index.sqlite` corruption:** unparseable `_index.sqlite` (disk error). Phase 1 recovery: rename to `_index.sqlite.corrupt.{ISO}`, re-create from markdown filesystem walk (decisions/ + proposals/ are source of truth; FTS5 is regenerable index). Telemetry `op_index_rebuild` emitted.
- **EC-05 — Conflict on conflicting decision (multi-conflict):** operator decision X conflicts with rule A AND rule B. V1: emit one warning per (decision, rule) pair. Debounce per-pair not per-decision (so user sees both warnings once).
- **EC-06 — Archive tier reactivation race:** `op:reinforce <archived-slug>` moves decision from MEMORY-archive.md → MEMORY.md. Concurrent preflight reading MEMORY.md mid-move: SQLite WAL ensures fts5 read sees consistent snapshot. Markdown read race: use atomic rename (write-tmp-then-rename) per BR-AN-04.
- **EC-07 — `user-profile.md` migration with conflicting fields:** existing decision `autonomy-preference-high.md` (from prior capture) and `user-profile.md` field `autonomy_preference: low`. Migration logic: skip field (preserve operator's own promotion). Log telemetry `op_migrate_skip` with reason. Do NOT silently overwrite.
- **EC-08 — Reserved prefix collision with hash:** sha256(email)[0..16] starts with `_` (reserved prefix). Probability ≈ 1/16 = 6.25%. Unacceptable. **Mitigation:** identity resolution rejects hash starting with `_` and re-hashes with salt `aioson-v1`. Salt is constant — deterministic for same email.
- **EC-09 — Windows path normalization on `~/.aioson/operators/{identity}/`:** `~` expansion on Windows resolves to `%USERPROFILE%`. Path separator backslash. Identity hashes are URL-safe (hex). No collision with Windows reserved names (`CON`, `PRN`, etc.) since hex hash starts with [0-9a-f].
- **EC-10 — Decay sweep during active session:** decay sweep runs at preflight. If sweep mid-archive when user triggers `op:show` for archived slug: WAL ensures read sees consistent state. Worst case: user sees decision moved to archive mid-display — re-run resolves.

## Open questions (deferred to Gate B — @architect)

The 5 DDs from sheldon manifest remain open. Two additional analyst-surfaced questions:

- **DD-A1** Atomic move semantics across phase boundaries — when Phase 5 archives a decision (active → MEMORY-archive.md), should the actual `decisions/*.md` file move too, or stay in `decisions/` with just the index pointer changing? Trade-off: filesystem locality vs link stability. _Impact: Phase 5 implementation._
- **DD-A2** FTS5 schema migration strategy — what's the V1 plan when `version_schema` bumps require `_index.sqlite` reset? Drop-and-rebuild from markdown filesystem (5s for 10k decisions) is simplest; incremental ALTER VIRTUAL TABLE is faster but FTS5-version-dependent. _Impact: V2 migration story (not blocking V1)._
