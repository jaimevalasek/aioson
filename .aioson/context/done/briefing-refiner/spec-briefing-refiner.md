## Key decisions

- 2026-06-08: Feature classification is SMALL — one primary operator role, no external integrations, complex artifact/state rules.
- 2026-06-08: `refinement-feedback.json` is the canonical machine-consumed feedback artifact; edited HTML/DOM is not source of truth.
- 2026-06-08: File write from `review.html` is progressive enhancement; export/copy/download fallback is required.
- 2026-06-08: Refinement artifacts live under `.aioson/briefings/{slug}/`, not `.aioson/context/`.
- 2026-06-08: New agent prompt must be template-first, then mirrored/synced to workspace.
- 2026-06-08: Modified approved briefings return to `draft` with `approved_at: null` instead of introducing `needs_reapproval`.
- 2026-06-08: `briefing-refiner` is an official available agent but not a mandatory workflow stage in V1.
- 2026-06-08: `@orchestrator` chose sequential execution for this feature; existing `.aioson/context/parallel/` lanes are from an older design-governance cycle and must not be reused for `briefing-refiner`.
- 2026-06-08: `briefing:approve`/`briefing:unapprove` now call the shared briefing registry helper — reason: refinement metadata and existing approval behavior must round-trip through one serializer.
- 2026-06-08: `briefing-refiner` was added to dossier canonical agent ids — reason: refinement sessions need to write Agent Trail/dossier events without schema rejection.
- 2026-06-08: `.claude` slash command stubs were added alongside AGENTS/CLAUDE references — reason: official agent invocation surfaces should stay consistent across supported clients.
- 2026-06-08: `conformance-briefing-refiner.yaml` was generated by @analyst after QA — reason: artifact validation in this MEDIUM repository requires the machine-readable AC contract even though the feature depth is SMALL.

## Edge cases handled

- Missing or malformed mandatory sections are blocked before serialization.
- Invalid section statuses, comment severities, schema versions and export methods are rejected.
- Cross-slug feedback is rejected.
- Stale source hash is rejected by default before automatic application.
- Generated HTML always includes download/copy fallback and treats direct File System Access save as progressive enhancement.
- Approved briefings with applied changes return to `draft` and clear `approved_at`.
- Declined feedback writes `refinement-report.md` with skipped changes while leaving `briefings.md` unchanged.
- Blocking items produce `resolve_blockers` instead of a product-ready next action.
- Existing `briefing:approve` and `briefing:unapprove` round-trip through the extracted registry helper.
