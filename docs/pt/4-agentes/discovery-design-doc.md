# @discovery-design-doc — Discovery e design doc sob demanda

> **Para quem é:** quem quer produzir ou revisar deliberadamente um discovery/design doc para uma decisão específica.

## Para que serve

O agente estrutura um problema vago, consolida evidência e pode criar um design doc/readiness quando esse documento é um entregável desejado.

Ele não é etapa pré-DEV obrigatória em SMALL ou MEDIUM. PRD aprovado e plano do Planner são as autoridades canônicas; design docs e readiness podem enriquecer, mas não bloquear por simples ausência.

## Quando invocar

- Um ticket ou ideia precisa de exploração antes de Product.
- O usuário pede explicitamente um design doc.
- Uma decisão técnica complexa se beneficia de alternativas e trade-offs documentados.
- O projeto já adota design docs como padrão local.

## Handoff

- descoberta que muda produto → Product/PRD;
- decisão que muda execução → Planner/plano;
- detalhe de implementação já aprovado → DEV.
